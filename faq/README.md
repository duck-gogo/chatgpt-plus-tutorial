## 支付与开通

### Q：国内 Visa / 银联信用卡能不能买 ChatGPT Plus？

经常有人拿国内办的 Visa 或 Mastercard 来试，结果还是被拒。OpenAI 的支付系统会按发卡地区（BIN 段）识别，只要是国内发卡，**即使卡面是 Visa / Mastercard 也直接拒付**，和你 IP 在哪、账单地址填哪、额度多高都没关系。

所以"我有一张 Visa"不等于"我能自己充值"。手里只有国内卡，更现实的做法是换不依赖海外信用卡的方式，而不是在同一张卡上反复试。展开见 [国内 Visa 能不能买 ChatGPT Plus](../articles/chatgpt-plus-without-overseas-card.md)。

### Q：没有国外信用卡，怎么用支付宝 / 微信充值 ChatGPT Plus？

没有海外卡也有几条路，主要这几种：

- **第三方自助代充平台**：用支付宝 / 微信付款，平台帮你把 Plus 开到你自己的账号上。省掉一整套支付排查，但要自己会挑平台。
- **iOS 内购**：主力用 iPhone / iPad、Apple ID 环境稳定，可以走 App 内订阅。

每种适合谁、要注意什么，详见 [主教程的方法对比](../README.md#summary)。核心是：先看自己属于哪类用户，再选路径，而不是硬刚国内信用卡。

### Q：国内信用卡为什么经常充值失败？

一般是支付风控、发卡地区、账单信息和网络环境共同作用。连续失败几次时别再硬试——先确认账号能否正常登录、网络是否稳定、支付页是否完整加载，再决定要不要换方案。详细排查见 [付款失败](../troubleshooting/payment-failed.md)。

## 第三方自助代充

### Q：第三方代充会不会要求给账号密码？

我把"是否要求账号密码"当成筛选红线。正常的自助流程**不应该**让你交出 ChatGPT 密码、邮箱验证码这类信息。对方张口要完整账号密码的，我直接放弃。像 <a href="https://plusgo.pro/?utm_source=github01" target="_blank" rel="noopener">PlusGO</a> 这种自助平台，提交的是一次性临时会话凭证（access_token，几小时后自动过期），不是密码。

### Q：一次性临时凭证和账号密码一样吗？

不是一回事，但它依然属于敏感信息。只在确认了网址、流程和订单都没问题时再提交，提交后保留订单号，遇到异常就走售后。

### Q：怎么判断一个代充平台靠不靠谱？

下单前至少过一遍这几条，能筛掉 90% 的草台班子：**不要账号密码、套餐周期和价格公开、有订单号可查、售后入口明确、不夸张承诺也不诱导高风险操作。** 完整清单见 [主教程的第三方平台自检清单](../README.md#checklist)。我个人把"不交密码"放在第一位。

### Q：通过代充平台开 Plus 会封号吗？

正规平台用企业级海外通道 + 干净住宅 IP，单纯因"代付动作"封号的概率很低。绝大多数封号是用户后期使用时网络节点不干净导致的，跟代充本身关系不大。挑平台时优先看运营年限、口碑和白纸黑字的售后条款。

### Q：充值成功后没看到 Plus 怎么办？

先做三件事：退出后重新登录；换无痕窗口或清缓存再看；确认提交的是不是当前正在用的账号。还是没变化就别反复重新下单，拿订单号联系售后。详见 [付款后账号状态不对](../troubleshooting/account-status.md)。

### Q：充值失败 / 已开通，退款怎么算？

分两种情况，下单前先看清平台条款：

- **充值失败、未开通成功** → 正规平台支持原路全额退款（PlusGO 是失败 100% 原路退款，通常 2 小时内到账）。
- **已成功开通且账号正常使用** → Plus / Pro 属于虚拟数字商品，已交付即视为消费完成，一般不支持七天无理由退款。

## 价格、续费与管理

### Q：到期后会自动续费吗？

看你选的方式。自己订阅或 iOS 内购通常有订阅管理入口；第三方自助代充一般更接近单次买断，到期前需要自己再操作一次。下单前先看清套餐周期、到期时间和退款规则。想彻底省去每月续费的麻烦，可以直接上长期套餐（比如 12 个月）。

### Q：开通之后怎么管理订阅？

- iOS 内购：在 Apple 的订阅管理里查看、取消、看账单。
- 自己用海外信用卡订阅：在账号设置的订阅页管理。
- 第三方自助代充：到期前回原平台再操作一次，留意续费说明。

不管哪种方式，建议留好订单号或凭证，方便后续查询和售后。

### Q：Plus 和 Pro 怎么选？

大多数普通用户先从 Plus 开始就够了。Pro 更适合每天高频重度使用 ChatGPT 或 Codex、经常被额度或任务长度打断的人。还不确定自己会不会高频使用，就别一上来买更贵的套餐。详细判断标准见 [主教程：Plus 用满后要不要看 Pro](../README.md#pro)。

## 使用与账号

### Q：开通之后还要注意网络环境吗？

要。充值只解决订阅状态，日常使用仍要稳定网络，尽量不频繁切换地区，也别用来路不明的公共节点。

### Q：新账号可以马上升级吗？

能否升级取决于账号状态、网络环境和支付方式。新账号建议先正常登录使用一段时间，确认邮箱、登录状态和基础功能都稳定后再升级。

### Q：失败后怎么保存排查信息？

养成随手记录的习惯，找售后或自己判断时都用得上：

- 失败时的**报错原文**（截图或抄下来）。
- 当时用的**充值方式**和**账号邮箱**。
- 走第三方平台的话，记下**订单号**和**下单时间**。
- 失败在哪一步：打开支付页、提交付款、还是付款后没到账。

留好这些，比凭印象说"就是失败了"好排查得多。对照报错排查见 [排障清单](../troubleshooting/)。
